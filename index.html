CareConnect (prototype)

A digital patient request and triage layer designed to complement existing call-light and care team communication workflows.

How it works

Capture → Prioritize → Route

Patients submit structured requests via QR or tablet with optional free text. The system standardizes intake, supports prioritization of time-sensitive vs routine needs, and routes requests by role (e.g., licensed vs support).

  • Structured buttons + guided follow-ups
  • Optional free-text note for context
  • Time-based visibility escalation for team awareness
  • Status feedback loop (ack / on my way / completed)
Designed for healthcare

Safety-first posture

Built to reduce ambiguity and relay burden without replacing established safety or emergency pathways.

  • Supports conservative prioritization
  • Access can be gated (e.g., signed QR + room active window)
  • Auditability for key events

Why it matters

Built from bedside reality to reduce alarm fatigue, unclear requests, and avoidable interruptions.

Reduce ambiguity

Standardized request capture helps staff understand what’s needed the first time.

Support equitable access

Designed for language and communication barriers, including Deaf and low-vision considerations.

Improve prioritization

Helps teams distinguish time-sensitive needs from routine requests without increasing alarm burden.
